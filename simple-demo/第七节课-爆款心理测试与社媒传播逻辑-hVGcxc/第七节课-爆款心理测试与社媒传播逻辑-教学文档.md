# 第七节课：爆款心理测试与社媒传播逻辑 教学文档

## 本节内容总结

本节课不是演示某个后台怎么点，而是围绕“怎样把一个心理测试类产品做得更容易传播”来拆方法。整节内容可以整理成三条主线：

1. 先把项目文档、任务计划和限制条件拆细，方便 `AI` 在开发时快速定位上下文，减少跑偏。
2. 再拆爆款心理测试为什么容易传播，包括用户心理、题目结构、结果页结构和传播链路。
3. 最后把方法落到平台选择和案例复盘上，判断什么样的测试更容易起量，什么样的内容容易做了也不传播。

如果后面要自己做心理测试、小程序测评、人格类内容产品，这节课最值得直接拿来复用的是：

- 文档怎么拆，才能让 `AI` 更稳定地协作开发
- 测试题怎么写，用户才愿意继续答完
- 结果页怎么写，用户才愿意截图、转发和讨论
- 不同平台上，内容包装方式应该怎么改

## 详细操作步骤

### 小tips：先把项目文档拆成 AI 容易读取的小文件
【视频回放时点00：05：00】

开始做产品前，先不要把所有规则都堆进一个总文档里。更稳的做法是把项目拆成多个独立文件，让 `AI` 可以按主题读取：

1. `prd/`：放项目概述、目标、功能范围
2. `frontend/`：放前端页面规划、布局要求、交互规则
3. `database/`：放数据结构、字段定义、约束条件
4. `task_plan.md`：放任务拆分、阶段安排、优先级
5. `notes.md` / `progress.md`：放每次开发后的记录和进度更新

这样做的作用是：

1. 让 `AI` 只读当前任务相关的文件，减少上下文浪费
2. 让每个限制条件都能落到具体文件里，避免开发时偏离目标
3. 方便后续多人协作，谁负责前端、后端、文档都更清楚

注意：
- 不要只留一个很长的总文档，否则后面越做越难定位重点。
- 每做完一轮任务，就更新一次笔记和进度文件，保证后续继续开发时能接上。

### 步骤1：先明确这节课要研究的对象和目标
【视频回放时点00：00：10】

正式进入案例前，先把本节课的核心范围记清楚：

1. 研究对象是“爆款心理测试”
2. 重点不是只看题目，而是看它背后的心理结构和传播结构
3. 目标不是做一个“会开发”的产品，而是做一个“用户愿意传播”的产品

这一页最关键的一句话是：爆款不是运气，而是可以复用的结构。后面的所有拆解，都可以围绕这个原则理解。

### 步骤2：先理解为什么心理测试天然适合传播
【视频回放时点00：10：00】

如果想做测试类产品，先把用户为什么愿意点开、做完、转发搞明白。这一页可以直接记成四个传播触发点：

1. `自我认同`
用户天然会对“我是什么类型的人”感兴趣。
2. `关系确认`
看到“我和 TA 合不合得来”这类问题，会自然带出社交互动。
3. `情绪出口`
焦虑、好奇、娱乐感都很容易被测试题承接。
4. `谈资价值`
结果如果能晒、能聊、能艾特朋友，就具备二次传播条件。

注意：
- 用户做测试，不是为了“答题”，而是为了拿到一个可以展示的身份结果。
- 如果结果不能形成表达欲，后续传播力就会明显下降。

### 步骤3：题目设计时，先守住“短、准、爽”三条底层原则
【视频回放时点00：20：00】

开始写题目前，先拿这三条原则逐项检查：

1. `短`
控制题量，避免题目过长。理想状态是单题读完时间很短，用户不会产生负担。
2. `准`
选项要贴近用户能立刻理解的生活场景，不要堆专业术语。
3. `爽`
用户每答一题都要感觉“这题在说我”，要有被理解、被点中的感觉。

题目写法上，优先用具体场景句，而不是抽象定义句。  
例如更适合测试传播的写法，是让用户在熟悉场景里快速代入，而不是让用户读到一条又学术又绕的描述。

注意：
- 题目越抽象，掉答题的概率越高。
- 一旦用户读题速度慢、代入感弱，后面就很难把人留到结果页。

### 步骤4：结果页一定要做成“用户愿意晒”的页面
【视频回放时点00：25：00】

测试做完以后，真正决定能不能传播的，往往不是题目，而是结果页。结果页至少要有这四块：

1. `标签名`
让用户一眼知道自己属于哪一类。
2. `一句解释`
用一句话说清楚用户当前的核心状态或特征。
3. `社交文案`
给出可以直接复制、转发、发圈的表达。
4. `结构化内容`
例如情绪描述、优势提醒、关系建议、分享按钮或截图区。

这里要记住一个判断标准：结果页如果做不到“哇”，用户就不会主动转发。

注意：
- 结果不要只给一个冷冰冰的分数。
- 如果没有可截图、可分享、可讨论的内容，传播链路会在结果页直接断掉。

### 步骤5：复盘案例时，按“选题、标题、结果、传播、转化”五个环节拆
【视频回放时点00：30：00】

不要只看一个测试“火了”，而要顺着完整链路拆它为什么火。复盘时可以直接按这五项做检查：

1. `选题`
是不是贴热点、贴关系、贴人格、贴节日等高感知主题。
2. `标题`
有没有让用户立刻代入，愿意点击。
3. `结果`
有没有形成可截图、可讨论的人格标签或结论。
4. `传播`
有没有设计评论区互动、朋友互测、二次转发的问题。
5. `转化`
有没有自然承接到下一步产品、服务或付费入口。

这一步很重要，因为它能帮助你区分“内容看起来热闹”和“结构上真的能传播”。

### 步骤6：做传播设计时，重点盯住四个关键时刻
【视频回放时点00：40：00】

把用户从“看到内容”到“主动转发”的过程拆开后，最关键的是这四个节点：

1. `看到标题的 3 秒内`
用户会不会想点。
2. `做到第 3 题左右`
用户会不会继续往下做。
3. `看到结果时`
用户会不会想截图。
4. `发出去以后`
评论区和聊天里会不会形成互动闭环。

做内容时，不要只想着“怎么发布出去”，而要反过来问自己：

1. 别人愿不愿意接着传
2. 用户做完后有没有自然的表达欲
3. 分享出去后，别人会不会被带着继续参与

### 步骤7：换平台时，不要直接复制同一套包装
【视频回放时点00：50：00】

同一个测试内容，放到不同平台上，表达方式要跟着平台受众改。可以先按这张对照表理解：

1. `抖音`
节奏快、情绪驱动强，前几秒一定要有钩子和冲突。
2. `小红书`
更重生活方式、人设和真实场景，标题和封面要更具体。
3. `视频号`
更适合把观点讲完整，适合建立熟人信任和清晰逻辑。
4. `公众号`
适合深度拆解、方法复盘和系统讲清楚，不适合写得太碎。

注意：
- 同一份内容不要原样分发到所有平台。
- 平台不同，用户对标题、节奏、表达深度的耐受度完全不同。

### 步骤8：最后用“热门 vs 不热门”的对照表做自检
【视频回放时点01：00：00】

如果一个测试做出来以后不火，不要先归因给“没有流量”，先按这张表反查差距：

1. `标题`
热门内容通常代入更强、冲突更明确。
2. `题目`
热门内容通常更直接，场景更具体，更容易读懂。
3. `结果`
热门内容更适合截图、分享、讨论。
4. `传播`
热门内容更容易带出评论区互动和二次分发。
5. `转化`
热门内容通常会顺着流量继续承接到下一步产品动作。

如果你的内容在这五项里连续几项都偏弱，就算发布出去，也大概率只是“有人看过”，很难形成真正扩散。

## 课后可直接复用的检查清单

如果准备自己做一个测试类产品，可以直接按下面顺序自查：

1. 是否先把 `PRD`、页面规则、数据结构、任务计划拆成独立文档
2. 选题是否同时具备自我认同、关系互动、情绪出口或谈资价值
3. 题目是否符合“短、准、爽”
4. 结果页是否具备标签、一句话解释、可复制文案和截图价值
5. 传播链路是否覆盖点击、继续答题、截图、互动四个关键时刻
6. 不同平台是否已经分别做了对应包装

只要把这几个检查点做扎实，这节课讲的核心方法就真正用起来了。
