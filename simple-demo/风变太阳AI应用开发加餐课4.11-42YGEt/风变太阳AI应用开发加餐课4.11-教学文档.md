# 风变太阳AI应用开发加餐课 4.11 教学文档

## 本节内容总结

这节加餐课主要是在上一节“前后端部署”基础上做补充和串讲，核心分成两条线：

1. 前端与轻量项目的 `Vercel` 部署流程
2. 后端项目的 `Go + 宝塔` 部署流程

课程里还穿插补充了这些关键内容：

1. 为什么要准备 `GitHub` 账号
2. 为什么部署前端时优先考虑 `Vercel`
3. 自定义域名如何接到 `DNSPod`
4. 环境变量应该放在哪里配置
5. `Go` 后端如何上传到服务器并在宝塔里运行

### 步骤2：先理解为什么前端部署优先讲 Vercel

【视频回放时点00：05：00】

课程先解释了 `Vercel` 的定位，可以直接记住这几条：

1. 代码推到 `GitHub` 后，`Vercel` 可以自动构建和部署
2. 默认支持 `HTTPS`
3. 每次 `push` 代码后都可以自动重新部署
4. 对个人项目和前端项目来说，免费额度通常够用

如果你的项目目前还是以前端为主，先走这条线会更快。

### 步骤3：先准备 GitHub 账号，后面所有自动部署都要用到

【视频回放时点00：07：00】

开始部署前，先确保你有可用的 `GitHub` 账号。课程里这里专门提醒了：

1. 很多开源项目、模板和教程都托管在 `GitHub`
2. 后面把代码接到 `Vercel` 时，也要先有仓库
3. 注册过程中如果遇到验证页面，按页面提示完成即可

如果当前还没有账号，先把注册流程走完，再继续后面的部署操作。

### 步骤4：部署前先学会在 GitHub 上搜索和参考开源项目

【视频回放时点00：10：00】

课程里这里还补了一条很重要的思路：不要只会从零开始写，先学会找参考项目。

操作思路是：

1. 先拆出自己要做的核心功能关键词
2. 到 `GitHub` 搜索更短、更准确的关键词
3. 先找可参考的项目结构、功能拆分和实现方式
4. 可以借鉴和参考，但不要直接把别人的开源项目原样拿来商用

这一步的重点不是立刻下载代码，而是建立“先找参考，再自己实现”的习惯。

### 步骤6：如果需要推代码，先把本机 SSH Key 准备好

【视频回放时点00：20：00】

如果后面要从本地把代码推到仓库，先确认本机已经有 SSH key。课程里这里演示了查看 `.ssh` 目录的方法：

1. 打开本机的 `.ssh` 目录
2. 找到公钥文件，例如 `id_ed25519.pub`
3. 后续把公钥加到 `GitHub`，就可以更顺畅地推送仓库

如果你已经配过 SSH，这一步主要是做确认；如果还没配过，就先把 SSH 登录链路补好。

### 步骤9：先记住 Vercel 部署的四步主流程

【视频回放时点01：10：00】

课程里把 `Vercel` 部署流程压缩成了 4 步，非常适合直接照着记：

1. 在 `Vercel` 点击 `Add New Project`，选择 `GitHub` 仓库
2. 让 `Vercel` 自动识别项目框架，保持默认配置
3. 到 `Settings -> Environment Variables` 添加环境变量
4. 点击 `Deploy`，等待生成 `vercel.app` 域名

如果你只是先把前端项目跑起来，这四步就是最短链路。

### 步骤10：如果项目有密钥或配置项，先在 Vercel 里补环境变量

【视频回放时点00：50：00】

项目里只要有密钥、回调地址、数据库连接等配置，就不要写死在代码里，而是放到 `Vercel` 环境变量里。课程里这里的页面能看到：

1. `NEXTAUTH_SECRET`
2. `NEXTAUTH_URL`

实际操作就是：

1. 打开项目设置中的 `Environment Variables`
2. 添加每个变量的键和值
3. 保存后重新部署，让新变量生效

![步骤10截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/42YGEt/work/samples/frame_3000.jpg)

### 步骤11：如果项目接了 Supabase，也要把连接关系和资源配置好

【视频回放时点01：00：00】

如果你的项目不只是静态页面，还用到了数据库或对象存储，这一节还补充了 `Supabase` 相关连接页。这里的重点是：

1. 确认 `Vercel` 项目已经连接到对应的 `Supabase` 资源
2. 确认环境变量会在 `Production / Preview / Development` 生效
3. 确认存储或数据库资源确实已经挂到当前项目

如果项目用了用户登录、图片上传、数据存储，这一步不要跳过。

### 步骤13：如果要绑定自己的域名，先回到 Vercel 添加域名

【视频回放时点00：40：00】

当默认域名可用后，就可以开始绑自己的业务域名。通常顺序是：

1. 打开 `Vercel` 项目的 `Domains`
2. 添加自己的域名
3. 看清 `Vercel` 给出的解析要求

后面真正让域名生效，还要去域名解析后台加记录。

### 步骤16：再看 Go 后端部署的四步主流程

【视频回放时点01：40：00】

课程后半段开始转到 `Go` 后端部署，并把步骤压成了 4 步：

1. 本地交叉编译 Linux 可执行文件
2. 通过宝塔文件管理器上传二进制文件和配置文件
3. 在宝塔里处理进程启动和自动重启
4. 在宝塔网站里配置反向代理到 Go 端口

如果你做的是前后端分离项目，这一条线就是后端上线的核心流程。

### 步骤19：如果项目依赖 MySQL 或其他数据库，要先把环境变量改成服务器可用的配置

【视频回放时点02：10：00】

中后段这里还补充了数据库相关配置。核心意思是：

1. 本地的 `SQLite` 或默认配置，部署到线上后通常要改掉
2. 改成线上真实可用的 `MYSQL_DSN` 或对应数据库连接串
3. 同时补上 `JWT_SECRET`、端口、项目目录等变量

如果后端要连数据库，这一步决定程序能不能真正跑起来。

## 本节结论

这节加餐课最重要的不是某一个单独页面，而是把“前端上线”和“后端上线”两条链路都串起来：

1. 前端用 `GitHub + Vercel + 域名解析 + 环境变量`
2. 后端用 `编译二进制 + 上传服务器 + 宝塔配置 + 域名/端口/代理`

如果你现在正处在“功能写出来了，但不知道怎么上线”的阶段，这一节最值得反复看的就是：

1. `Vercel` 的 4 步部署流程
2. 域名解析与 `DNSPod` 配置
3. `Go` 后端的 4 步部署流程
4. 宝塔里的项目配置、环境变量和代理规则
