# 风变太阳AI应用开发 4.8 教学文档（细化修订版）

## 本节内容总结

本节课分成两条并行主线：

1. 先用概念页说明，为什么后端部署通常有两条路：
   `Vercel` 适合快速上线
   `宝塔 + 云服务器` 适合需要更多控制权的后端项目
2. 再重点演示 `宝塔` 这一条链路，包含：
   购买云服务器
   选择镜像
   初始化宝塔面板
   上传后端文件
   修改环境变量
   调整文件权限
   在宝塔里创建并启动 `Go` 项目
   通过计划任务或辅助工具补足后续运维动作

这版文档会保留概念页面，同时把宝塔中间的调试和确认动作写细。

### 步骤2：先理解 Vercel 和宝塔各自适合什么场景

【视频回放时点00：07：00】

正式动手前，先建立一张全景图：

1. `Vercel` 更适合 `Next.js / Node.js` 这一类快速上线场景
2. `宝塔 + 云服务器` 更适合 `Go / Java / Python` 这类需要完整服务器控制权的后端
3. 如果你希望国内访问速度更稳、环境更可控，服务器方案更值得考虑

这一步的作用不是让你二选一，而是让你知道为什么后面要详细讲宝塔。

### 步骤3：先理解为什么 Go 项目更适合放到宝塔上

【视频回放时点00：55：00】

如果你的后端是 `Go`，更适合放到自己的服务器上跑。原因可以直接记成这几条：

1. `Go` 编译之后通常就是一个二进制文件
2. 放到自己的服务器上更灵活，不容易被平台限制
3. 文件管理、进程守护、`Nginx`、`SSL` 这类动作都能在宝塔里做
4. 对个人项目来说，宝塔的可视化操作能显著降低命令行门槛

### 步骤6：镜像这里直接选 Debian

【视频回放时点00：20：00】

进入镜像选择时，把系统切到：

1. `Debian`

然后从列表中选择合适版本。当前演示中展开了多个 `Debian 64 位` 选项。  
如果你后面要装宝塔、跑后端、做文件管理，`Debian` 这一类常见 Linux 发行版会比较稳。

选镜像时建议顺手检查：

1. 版本号
2. 位数是否是 `64位`
3. 镜像默认磁盘大小

### 步骤12：环境变量不要写死在代码里，要在服务器文件里补齐

【视频回放时点01：05：00】

课程里明确强调了环境变量和数据库连接这一块。  
不管是本地开发，还是线上服务器，都应该把密钥和数据库连接信息放进环境变量，而不是直接写死在代码里。

当前概念页给出的原则是：

1. 密钥不上 Git
2. 数据库连接走环境变量
3. 线上环境和本地环境的变量要分别管理

![步骤12截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/RUWNwF/work/frame_3000.jpg)

### 步骤13：在宝塔文件编辑器里修改 `.env.local` 或对应环境文件

【视频回放时点01：10：00】

真正进入调试时，会在宝塔文件编辑器里直接改环境变量文件。  
当前画面里能看到的关键字段包括：

1. `DATABASE_DSN`
2. `JWT_SECRET`

这一步要做的是：

1. 把数据库连接字符串填对
2. 把密钥补齐
3. 保存文件

注意：

1. 变量名不要拼错
2. 数据库地址、端口、库名、账号、密码都要对齐
3. 保存之后，如果项目没自动生效，后面还要回到项目管理页重新加载或重启

### 步骤14：如果二进制文件无法执行，就到权限页里处理

【视频回放时点01：32：00】

当前画面里打开的是 `api-linux` 的文件属性页，可以看到：

1. 所有者权限
2. 用户组权限
3. 公共权限
4. `应用到子目录`
5. `应用` 按钮

如果你发现程序没有执行权限，就在这里把权限补正确，再保存。

这一类问题非常常见，不能省略。

## 本节结论

如果你当前做的是需要后端、数据库、文件管理和服务器控制权的项目，这节课最应该掌握的是这一条链路：

1. 先理解为什么要选宝塔
2. 在腾讯云把云服务器买对
3. 用 `Debian` 初始化服务器环境
4. 进入宝塔完成面板初始化
5. 上传 `Go` 后端文件
6. 配 `.env`
7. 调整权限
8. 在 `Go项目` 页确认运行状态
9. 用 `计划任务` 处理后续自动化动作

概念页不要跳过，因为它负责解释“为什么这样做”；  
宝塔操作步骤也不要省略，因为真正卡住人的，往往就是中间这些调试细节。
