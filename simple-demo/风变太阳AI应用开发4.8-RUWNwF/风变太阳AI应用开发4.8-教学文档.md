# 风变太阳AI应用开发 4.8 教学文档

## 本节内容总结

本节课聚焦“项目上线后的后端部署”。核心目标不是继续堆前端页面，而是把已经完成的应用真正发布出去，并补齐部署、数据库、文件存储和服务器工具链。

本节重点包括：

1. 理解 `Vercel` 是什么，以及为什么它适合 `Next.js` 项目快速上线
2. 理清一条最轻量的部署链路：`代码放 GitHub -> 导入到 Vercel -> 创建 Postgres / Blob -> 配环境变量 -> 一键部署`
3. 认识 `Vercel` 和 `宝塔 + 云服务器` 两套方案的适用场景差异
4. 演示如何在项目目录中拆分 `frontend / backend / docs` 等结构，方便后续维护
5. 演示如何在宝塔面板中处理数据库与服务器端工具
6. 演示如何在服务器侧准备“视频提取语音”这类辅助工具，为后续内容生产或自动化处理做准备

### 步骤1：先明确本节课的目标是“补上后端部署”

1. 低运维的快速上线方案：`Vercel`

### 步骤2：先认识 Vercel 是什么

【视频回放时点00：10：00】

先建立一个基本认识：`Vercel` 的定位，就是把代码推到 `GitHub` 后，由平台自动帮你完成构建、部署和 `HTTPS` 配置。  
如果你的项目是 `Next.js`，那么它和 `Vercel` 的组合会非常顺手，尤其适合快速上线和做 MVP 验证。

这一部分要记住的结论有三点：

1. 不需要自己折腾证书，默认就能用 `HTTPS`
2. 每次代码 `push` 之后，可以自动重新部署
3. 分支还能生成预览链接，方便测试和验收

![步骤2截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/RUWNwF/work/frame_600.jpg)

### 步骤3：把部署主流程先记成固定动作链

【视频回放时点00：30：00】

在真正动手前，先把最重要的部署动作链记住。后续无论是做笔记、自己复盘，还是带着项目重新走一遍，都优先按这个顺序执行：

1. 代码放到 `GitHub`
2. 把仓库导入到 `Vercel`
3. 在 `Vercel` 控制台创建项目依赖的资源
4. 按项目需要创建：
   `Postgres`
   `Blob`
5. 配置环境变量
6. 一键部署并验证结果

如果你的目标是自己使用、快速上线、减少运维，这一条链会非常合适。

### 步骤4：根据项目目标决定是否直接选 Vercel

【视频回放时点00：30：00】

如果当前项目满足下面几类诉求，就可以优先考虑直接走 `Next.js + Vercel Postgres + Vercel Blob` 这条组合：

1. 项目主要给自己使用
2. 当前是图文笔记、内容型产品，后端逻辑不算特别重
3. 想尽快上线，先验证产品是否有人用
4. 想把运维工作压到最低

当你的目标是“先让产品跑起来”，而不是“先把服务器体系搭到最复杂”，这套组合通常更省时间。

![步骤4截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/RUWNwF/work/frame_1800.jpg)

### 步骤5：在项目目录里把前后端和文档拆开

【视频回放时点01：00：00】

进入项目落地阶段后，目录结构不要混在一起。至少先把核心部分拆清楚，后续调试和部署都会更省心。

当前演示里可以看到的目录拆法包括：

1. `backend`
2. `frontend`
3. `docs`
4. `README.md`
5. `DEPLOYMENT.md`

这样拆的好处是：

1. 前端和后端职责清楚
2. 文档和代码不混在一起
3. 后续接手、部署、排查问题都更容易定位

![步骤5截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/RUWNwF/work/frame_3600.jpg)

### 步骤6：在宝塔面板里处理数据库相关工作

【视频回放时点01：00：00】

如果项目不只走 `Vercel`，还需要服务器侧能力，那么数据库配置就会进入实际操作环节。  
这时可以在宝塔面板中进入 `数据库` 模块，查看现有库、添加数据库，或者为后端项目准备连接信息。

这一段的重点不是死记某个按钮，而是记住服务器侧数据库处理通常要做这几件事：

1. 确认数据库类型
2. 创建或检查业务库
3. 给后端准备连接参数
4. 把数据库信息同步回项目环境变量

![步骤6截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/RUWNwF/work/frame_3600.jpg)

### 步骤7：在宝塔环境里准备服务器工具

【视频回放时点01：30：00】

当课程进入服务器工具链演示时，重点已经从“单纯部署页面”扩展到“让服务器能承担更多自动化任务”。  
这时需要把工具运行界面、输入输出目录、依赖命令都先准备好。

当前演示画面里展示的是一个“视频提取语音”工具界面，它的准备动作包括：

1. 选择视频文件或填写视频链接
2. 指定输出目录
3. 配置 `FFmpeg`
4. 勾选是否启动语音转文字
5. 选择模型、语言与导出格式

如果你的项目后续要做内容处理、音频转写或自动化整理，这类工具链准备好之后会非常省时间。

### 步骤8：回到 Vercel 检查部署结果

【视频回放时点02：00：00】

部署完成后，不要只看“已经提交代码”就结束。一定要回到 `Vercel` 控制台检查部署结果，确认实际线上构建有没有成功。

这里重点查看：

1. `Deployments` 列表里是否出现最新记录
2. 状态是否是 `Ready`
3. 当前分支是否正确
4. 最近提交说明是否和本次改动一致

这一步的作用是把“代码提交成功”和“线上真正可用”区分开来。

![步骤8截图](/Users/shihui/Desktop/玩玩编程/生产教学文档/simple-demo/RUWNwF/work/frame_7200.jpg)

### 步骤9：最后再判断是选 Vercel，还是选宝塔 + 云服务器

【视频回放时点02：15：00】

当你把整节课看完，最终要能自己做一个方案判断，而不是只记住工具名字。

可以这样理解：

1. 如果你做的是 `Next.js` 项目、想快速上线、运维越少越好，优先选 `Vercel`
2. 如果你需要更强的服务器控制权，或者后端是 `Go / Java` 这一类更重的服务，`宝塔 + 云服务器` 会更合适
3. 如果你在意国内访问体验，也可以优先评估服务器方案

这一页不是单纯做对比，而是帮助你在项目落地时少走弯路。

## 本节结论

如果当前阶段的目标是“让产品先活起来”，就先把 `GitHub + Vercel + 数据存储 + 环境变量 + 部署检查` 这一条主线走通。  
如果后面需要更复杂的后端能力、服务器工具链或更高的控制权，再逐步把 `宝塔 + 云服务器` 这一条方案补上。
