# 截图与点击标注测试记录

## 1. 测试目的

验证以下方案是否可行，并可写入 PRD：

- 不按固定间隔抽帧，而是在步骤窗口内寻找动作前/动作后截图
- 用视觉差分辅助确定关键变化区域
- 用 OCR 提供控件文本锚点
- 用“多信号融合 + 低置信降级”生成点击标注建议

## 3. 测试方法

### 3.1 连续帧抽取

从同一段真实回放中，连续抽取 10 张 1fps 帧，观察步骤窗口内的变化。

### 3.2 帧间差分

对相邻帧做灰度差分，统计：

- `mean_diff`
- `changed_px`

结果如下：

- `frame_08 -> frame_09`: `mean_diff=1.05`, `changed_px=6311`
- `frame_09 -> frame_10`: `mean_diff=1.14`, `changed_px=7608`

初步结论：

- `frame_09 -> frame_10` 是这一段变化最明显的相邻帧对
- 可以把 `frame_09` 作为动作前候选，把 `frame_10` 作为动作后候选

### 3.3 教学区域过滤

直接对整图做 diff，变化框会被浏览器地址栏、桌面边缘和压缩噪声污染，因此增加教学区域 ROI，排除：

- 浏览器地址栏
- 桌面 Dock
- 摄像头角标
- 页面外边缘

过滤后，热点主要落在：

- “修改名称”标题附近
- “小程序名称”输入区域
- 输入框内文本变化区域
- “下一步”按钮附近的局部区域

### 3.4 OCR 验证

使用 macOS `Vision` 对 `frame_10` 做 OCR，成功识别出关键控件文案，包括：

- `修改名称`
- `小程序名称`
- `王橙子心理测试`
- `下一步`
- 说明文字与错误提示

这说明：

- 不依赖额外 OCR 大包，也能获得可用的文本锚点
- 可以把 OCR 结果与差分热点绑定，形成“控件文本 + 变化区域”的复合证据

## 4. 点击标注测试结果

### 4.1 成功点

- OCR 识别出的 `小程序名称` 和输入内容 `王橙子心理测试` 与差分热点位置一致
- 在教学区域过滤后，热点集中到弹窗内容区，而不是整图噪声
- 这足以生成一个低到中等置信度的候选点击对象：
  - 候选对象：`小程序名称输入框`
  - 证据来源：`ocr + visual_diff`
  - 输出形式：文字锚点或疑似点击框

### 4.2 失败点

- 该测试片段没有稳定拿到可复用的鼠标轨迹
- 仅靠整图 diff 会得到几乎覆盖整张图的粗框，不可直接用
- 没有语音动作词参与时，点击点只能做到“疑似目标”，不能假装精确

### 4.3 结论

点击标注链路可以落地，但必须分置信度层级：

- 高置信：`cursor + OCR + diff`
- 中置信：`OCR + diff`
- 低置信：只有单一弱证据，必须输出“请人工确认”

## 5. 对 PRD 的直接结论

这次测试支持以下产品策略：

- 每个步骤优先找动作前截图和动作后截图，而不是平均抽帧
- 视觉差分只能作为候选信号，不能单独决定截图和点击框
- 必须先过滤非教学区域，再做候选热区分析
- OCR 是可行的文本锚点来源
- 缺少光标时，允许降级为“疑似点击位置 + 人工确认”

## 6. 是否测试成功

结论：`成功，但为有限成功`

成功的部分：

- 动作前/动作后截图候选可从真实视频中提取
- OCR 与差分能够在真实课程片段上形成联合证据
- 点击标注可以输出可审核的候选结果，而不是纯猜测

仍需后续增强的部分：

- 鼠标轨迹稳定提取
- 语音动作词与界面变化的时间对齐
- 更稳的组件级 UI 识别
