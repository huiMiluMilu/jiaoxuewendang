01 · VIBE CODING

从“写代码”变成“指导 AI”

我们把 AI 应用开发称为 Vibe Coding（氛围编程）：用自然语言开发软件，而不是自己手写每一行代码。你的角色从单纯的开发者，变成产品设计者和 AI 指导者。

以前 · 手写开发

需要自己一行一行写代码，把功能实现出来。

→
现在 · Vibe Coding

告诉 AI：“帮我做一个 XX 功能的网页或 App”，AI 生成代码并协助完成开发。

AI Coding 的基础流程

先和 AI 对话梳理产品需求文档（PRD），再让 AI 根据 PRD 制定开发计划，最后开始开发。

02 · PRODUCT TYPES

常见产品形态

生活中的软件像不同类型的“店铺”，会根据使用场景选择不同形态。

01网页 Webpage

运行在浏览器里的程序界面。输入网址即可打开，不需要安装。

  • 浏览器把背后的代码“翻译”成文字、图片和按钮。
  • 核心特点：即开即看，关闭就走。

02手机原生应用 Native App

从 App Store 或安卓应用市场下载并安装。

  • 能更顺畅调用摄像头、麦克风、GPS。
  • 例子：微信、抖音、支付宝。

03桌面应用 Desktop App

安装在 Windows 或 Mac 电脑上的软件。

  • 性能最强，适合复杂创作和大型任务。
  • 例子：Word、飞书、PS、PR、Steam 游戏。

04Web App

看起来像 App，实际仍通过浏览器打开，不用安装。

  • 兼顾网页免安装和 App 的流畅交互。
  • 例子：腾讯文档、Figma、在线版 PS、网页点单。

05小程序 Mini Program

藏在微信、支付宝、抖音等“超级 App”里的微型应用。

  • 用完即走，不占太多存储。
  • 例子：喜茶点单、微信小游戏、美团微信端。

03 · STATIC / DYNAMIC

静态应用 vs 动态应用

任何产品形态，都可以从技术原理上再分成静态和动态。

静态 Static 📄

  • 内容基本不变，像一张电子传单。
  • 所有人打开看到的内容都一样。
  • 适合个人介绍页、产品宣传页、活动规则页。

动态 Dynamic 🔄

  • 根据不同的人、时间和操作展示不同内容。
  • 背后有数据库这个“隐形仓库”。
  • 例子：淘宝推荐、小红书内容、手机银行余额。
记忆法：静态像路边海报，谁看都一样；动态像餐厅定制点单，会根据你的要求做出不同结果。
04 · FRONTEND / BACKEND

前端与后端怎么配合？

开发一个完整的软件，通常需要两种技术配合。继续用“去餐厅吃饭”来理解。

前端 Frontend 🍽️

  • 用户看得到、点得到的界面、文字、图片、按钮和动画。
  • 负责界面好看、好用，并接收用户操作。
  • 像餐厅的大堂、菜单和服务员。

后端 Backend 👨‍🍳

  • 藏在界面背后的服务器和数据库。
  • 负责业务逻辑、密码验证、数据存储等核心工作。
  • 像餐厅的后厨和食材仓库。

一次点赞的完整流程

1 · 前端响应

用户点击心形按钮，前端立刻把空心图标变成红心，并告诉后端谁点了赞。

2 · 后端记录

服务器收到信息，在数据库里给对应用户记录“点赞 +1”。

3 · 闭环展示

后端返回记录成功，下次打开时，前端把最新点赞结果展示出来。

✨ 如果一个人既能搞定前端大堂，又能处理后端厨房，就是“全栈工程师 Full Stack”，相当于自己开了一家店。
