# 同事怎么和 AI 对话来用这个仓库

这份文档是给“已经拿到仓库地址、想直接让 AI 帮自己跑”的同事准备的。

适用场景：

1. 已经克隆了仓库
2. 已经运行过 `bash install.sh`
3. 正在用任意一个支持“读本地仓库 + 跑命令”的 AI 编程工具

不要求一定是 `Codex`。  
只要 AI 能读仓库文件、执行命令、编辑文件，就可以照着这里的话术直接用。

## 一、先给 AI 的第一句话

如果同事第一次打开这个仓库，最推荐先发这段：

```text
请把这个仓库当成一个“根据课程回放生成教学文档”的工作流来使用。
先阅读 video-teaching-doc-skill/SKILL.md，
再结合 QUICKSTART.md、UNIVERSAL_SETUP.md 和 scripts 目录里的脚本工作。
目标是：根据我给的视频链接，先转写，再整理成教学文档 Markdown；
文档要按讲解顺序写，每一步前加【视频回放时点xx：xx：xx】；
如果适合，再补截图；如果环境支持，再发布到飞书。
先不要跳过转写，先确认课程主线，再开始写文档。
```

这段话的作用是：

1. 先让 AI 读规则
2. 告诉 AI 这是一个现成仓库，不是从零瞎猜
3. 明确“先转写，再成文，再补图”的顺序
4. 明确每个新视频要单独放一个文件夹，后续方便整目录清理

建议顺手再补一句：

```text
每个新视频请单独创建一个文件夹，目录名根据视频标题或课程主题命名。
最终文档和中间产物都放在这个目录里，方便我确认没问题后直接删掉整个文件夹。
```

## 二、最常用的 4 组对话模板

### 模板 1：先出一版 Markdown 教学文档

```text
请用这个仓库帮我处理一个课程回放链接：
<把视频链接放这里>

要求：
0. 每个新视频请单独创建一个文件夹，目录名根据视频标题或课程主题命名
1. 先判断这节课是偏操作演示，还是偏概念讲解
2. 先抽音频并转写
3. 按讲解顺序整理成教学文档 Markdown
4. 每一步前加【视频回放时点xx：xx：xx】
5. 如果是概念课，就写成讲义式教学文档
6. 如果是操作课，就写成详细步骤式教学文档
7. 如果是讲解课，不要写“步骤1、步骤2”，改用“一、二、三、”
8. 如果是在某个大标题下面展开小点，再用“1、2、3”

先给我本地 Markdown 版本，不用急着发飞书。
```

适合：

1. 先验证课程方向
2. 先看文案对不对
3. 不想一开始就被截图和飞书卡住

### 模板 2：在 Markdown 基础上继续补截图

```text
继续基于刚才这版文档补截图。

要求：
1. 截图必须和当前步骤内容严格对应
2. 如果是概念页，截图要对应当前讲到的那一页
3. 如果是操作页，只要有明显点击或页面变化，就补截图
4. 不要把图片整体错位
5. 如果某一页不适合截图，就换一张更准确的帧

先更新本地 Markdown，再给我看。
```

适合：

1. 文案已经基本确认
2. 开始进入图文版

### 模板 3：直接发飞书版

```text
把当前这版教学文档发布到飞书云文档。

要求：
1. 保持现在的标题和步骤结构
2. 图片按正文顺序插入
3. 发布完成后把飞书链接、本地 Markdown 路径、发布结果 JSON 路径都告诉我
4. 如果图片上传失败，先排查再重发，不要直接结束
```

适合：

1. 已经看过本地版
2. 想交给老师、运营、教研同事直接看

### 模板 4：继续细化当前视频

```text
继续细化这条视频的教学文档。

要求：
1. 保留当前主线，不要推翻重写
2. 补更多中间截图
3. 把过于提纲式的内容扩写成更像正式教学讲义的语言
4. 如果开头有方法论或概念页，要写得更完整
5. 新产出请单独存成“细化修订版”，不要覆盖上一版
6. 细化完成后再发一版飞书
```

适合：

1. 第一版已经对了
2. 想继续把它打磨成更正式的版本

## 三、不同课程类型，怎么和 AI 说更准

### 1. 如果是操作演示课

直接这样说：

```text
这节课是操作演示课，不是概念课。
请按实际操作顺序写成详细步骤，
凡是有鼠标点击、表单输入、页面变化、弹窗出现/消失的地方，都尽量补截图。
不要写成总结，要写成学员可以直接照着做的教学文档。
```

### 2. 如果是概念讲解课

直接这样说：

```text
这节课主要是概念讲解，不是后台操作演示。
请写成讲义式教学文档，
重点保留本节课讲了什么、为什么这样做、有哪些可复用的方法。
截图要对应当前讲到的概念页，不要硬写成点击教程。
标题请用“一、二、三、”，不要写“步骤1、步骤2”。
如果某个大标题下面需要展开小点，再用“1、2、3”。
```

### 3. 如果是“概念 + 操作混合课”

直接这样说：

```text
这节课是概念讲解和实操混合。
请把概念页保留成讲义式说明，
把操作部分写成步骤式说明，
不要把纯概念页硬改成点击步骤，也不要把操作过程写成旁白总结。
概念部分标题用“一、二、三、”，真正的实操部分再用“步骤1、步骤2”。
```

## 四、如果 AI 第一次做偏了，第二轮怎么纠正

下面这些是最常用的“纠偏话术”，可以直接复制。

### 1. 文案太粗

```text
这版太粗了，请继续细化。
不要只写提纲，要把每一步扩写成更像正式教学文档的表达。
```

### 2. 先看图后写文，方向偏了

```text
思路不对。
先把音频转成文字，再根据老师讲解顺序整理文档，
最后再插入对应截图。
不要先看图猜内容。
```

### 3. 图片错位

```text
图片和步骤错位了。
请逐张复核截图内容，再重新对应到正确步骤。
不要整体往前或往后平移一位。
```

### 4. 概念课被写成操作课

```text
这节课主要是概念讲解，不是操作演示。
请改成讲义式教学文档，保留方法论和概念拆解。
```

### 5. 操作课被写成总结

```text
这节课主要是实操，不是纯总结。
请按操作顺序重写，详细说明点哪里、填什么、注意什么。
```

## 五、推荐给同事的完整对话流程

如果同事不知道怎么一步步聊，最稳的是按这个顺序：

### 第一句

```text
请先阅读 video-teaching-doc-skill/SKILL.md，
再用这个仓库帮我处理一个课程回放链接：
<视频链接>
先转写并确认课程主线，再生成本地 Markdown 教学文档。
```

### 第二句

```text
继续把这版文档整理得更像正式教学文档。
每一步前加【视频回放时点xx：xx：xx】。
```

### 第三句

```text
继续补截图，要求截图和步骤严格对应。
```

### 第四句

```text
把当前版本发到飞书，并把飞书链接给我。
```

## 六、推荐同事顺手告诉 AI 的补充信息

如果同事自己已经知道这节课是什么类型，提前告诉 AI，效果会更稳：

```text
补充信息：
1. 这节课主要是操作课 / 概念课 / 混合课
2. 我更希望先要 Markdown 还是直接要飞书版
3. 我更看重文案准确，还是更看重截图完整
4. 如果截图和文案冲突，请先以转写和课程主线为准
```
